Что я сделаю на вашем проекте

Восстановлю документацию там, где её нет. Реверс-инжиниринг кода, схем БД, поведения системы, интервью с разработчиками. Превращаю всё это в нормальные SRS, ТЗ, схемы интеграций. На медтех-проекте восстановил документацию по 30% недокументированной функциональности.

Напишу новую документацию с нуля. BRD, FR, NFR, ТЗ, ЧТЗ, ADR. Доку пишу так, чтобы по ней реально могли работать разработка, тестирование и эксплуатация — а не «для галочки».

Спроектирую API и интеграции. OpenAPI/Swagger-спецификации, схемы обмена, Kafka-топики, контракты сообщений, версионирование, обратная совместимость. REST, gRPC, SOAP, Webhook, OAuth 2.0, OIDC, mTLS.

Разберу архитектуру. Что у вас работает, что мешает, где границы микросервисов, какие зависимости, где узкие места. Нарисую C4-модель, BPMN, Sequence-диаграммы, ERD — как инструмент обсуждения и принятия решений.

Декомпозирую бизнес-задачу до постановок разработке. User Stories с acceptance criteria в формате Given-When-Then. Edge-кейсы и негативные сценарии прорабатываю заранее — чтобы QA не возвращал баги пачками после релиза.

Спроектирую модель данных. ERD, нормализация, индексы, партиционирование, миграции. PostgreSQL, Oracle, MS SQL Server. SQL уверенный — CTE, оконные функции, EXPLAIN, оптимизация запросов.

Соберу прототип. Если нужно быстро проверить гипотезу — Python/FastAPI на бэкенде, SQL для модели данных, прототип UI в Figma. Не претендую на роль разработчика, но рабочий MVP под обсуждение с бизнесом сделаю.

Возьму коммуникацию между бизнесом, разработкой, QA, безопасниками. Переведу с языка на язык, согласую решения, зафиксирую договорённости письменно — чтобы потом не было «а мы думали наоборот».

Доведу проект до приёмки. Acceptance criteria, тест-сценарии, протоколы приёмо-сдаточных испытаний, передача в поддержку.
